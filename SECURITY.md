# Security Policy (IX-Vibe)

## Supported Versions
This repository is an evaluation kit. Security issues are handled case-by-case.

## Reporting a Vulnerability
If you discover a security issue (e.g., in scripts, dependency handling, or unsafe instructions):
1) Do **not** open a public issue with exploit details.
2) Contact the author (Bryce Lovell) with:
   - A clear description of the issue
   - Steps to reproduce
   - Impact assessment
   - Suggested fix (if available)

## Scope Notes
- This repo should not include secrets or credentials.
- Do not store proprietary test data in the repo unless explicitly intended.
